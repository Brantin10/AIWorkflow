# Context Bundle

Assemble vault knowledge into focused context for the current task.

## Load Existing Bundle

```bash
# Via API
curl -s "http://localhost:3211/api/vault/bundles?project=ProjectName"

# Via file
# Read: C:\Users\oliver\Obsidian\DevVault\01-Projects\bundles\bundle-project-name.md
# Or read: .ai-hub/vault-context.md (auto-generated by Orchestrator)
```

## Create New Bundle (Auto)

```bash
# Orchestrator auto-generates bundle from matching vault notes
curl -X POST http://localhost:3211/api/vault/bundles \
  -H "Content-Type: application/json" \
  -d '{"project": "ProjectName"}'
```

This automatically:
1. Reads the project note from `01-Projects/`
2. Finds matching ADRs by stack tags
3. Finds matching coding skeletons
4. Finds project research notes
5. Finds relevant debug playbooks
6. Assembles into a single context document
7. Saves to vault + project's `.ai-hub/vault-context.md`

## Create Bundle Manually

1. Read project note: `01-Projects/project-name.md` — get stack and tags
2. Search `03-Patterns/architecture/` for ADRs with matching stack
3. Search `02-Skills/coding-skeletons/` for skeletons matching the stack
4. Search `04-Research/project-name/` for project research
5. Assemble relevant content into working context
6. Save to `01-Projects/bundles/bundle-project-name.md`

## When to Create Bundles

- Starting a new project
- Before a major refactor
- Before working on an unfamiliar part of the codebase
- When switching between projects (to reload context)
- Before complex multi-file changes
